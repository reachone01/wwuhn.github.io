IIS站点及设置项备份与恢复

在准备重装操作系统前，我们可以先对IIS站点进行备份（该备份也会包含站点的相关设置），当操作系统安装完成后再进行还原。就可以减少系统重装好后一步一步手工配置站点的过程，并且此时我们不用关心站点下面做了哪些设置，因为进行备份还原后这些设置也会随之被还原。当然，如果IIS的配置相关文件被损坏，如果有对应的备份文件，也可以起到快速恢复的作用。

下面看一下具体的操作方法：

一、IIS6（在管理工具→IIS→帮助可以查看版本，win7的是IIS7）

【备份】：打开IIS，在计算机名称处点击鼠标右键，选择“所有任务”->“备份/还原配置”，在弹出的窗口中选择“创建备份”，填写备份名称、选中“使用密码加密备份”并设置备分密码。

备份好的文件存放于“C:\WINDOWS\system32\inetsrv\MetaBack”目录中，将其进行备份。

注意：一定要勾选中“使用密码加密备份”，否则重装系统后进行还原时将会出现“无效签名”的错误而无法还原成功。

备份的文件及文件目录如上图

【还原】：

1、首先将备份的文件拷贝至“C:\WINDOWS\system32\inetsrv\MetaBack”目录，然后在IIS计算机名称处点击鼠标右键，选择“所有任务”->“备份/还原配置”，此时系统会列出之前备份的备份，选中要还原的备份，点击“还原”按钮，输入密码即可。

看看之前的应用程序池、所有网站端口号、文件路径、主机头、站点中配置的主目录、日志记录属性、MIME类型等配置，是不是和原来完全一样呢？^_^

二、IIS7及以上版本的操作方法：

【备份】：在CMD中执行以下命令：

::进入IIS路径

cd /d c:\windows\system32\inetsrv

::执行备份命令

appcmd add backup "备份名称"

上面命令执行成功后，会在“C:\Windows\System32\inetsrv\backup”目录生成IIS的备份文件夹，将其进行备份。

【还原】：

首先将备份的文件拷贝至“C:\Windows\System32\inetsrv\backup”目录，然后在CMD中执行以下命令进行还原操作：

::进入IIS路径

cd /d c:\windows\system32\inetsrv

:: 列出所有备份(用于查看当前存在的备份)

appcmd list backup

::执行备份命令

appcmd restore backup "备份名称"

说明：执行该还原操作时可能会遇到报“ERROR ( hresult:800710d8, message:命令执行失败。对象标识符没有表示一个有效的对象。)”的错误提示。网上找了很久没有找到解决办法，后来发现还原是成功的，但IIS服务没有启动，只要手动启动IIS服务就可以了，而此时查看IIS站点，都已经还原好，网站也一切运行正常。
